# @ResponseBody Best Practices

## 1. Understanding @ResponseBody

`@ResponseBody` tells Spring MVC to serialize the return value directly to the HTTP response body (JSON/XML), bypassing view resolution.

- `@RestController` = `@Controller` + `@ResponseBody` (implicit for all methods)
- Use `@ResponseBody` explicitly only with plain `@Controller`

## 2. Custom @ResponseBody Annotation

### When to Use

When you need **custom wrapping behavior**, **logging**, **header manipulation**, or **conditional serialization** for specific endpoints.

### Implementation Pattern

#### Step 1: Create the Annotation

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface MyResponseBody {
}
```

#### Step 2: Create Response Wrapper DTO

```java
public record ApiResponse<T>(
    LocalDateTime timestamp,
    int status,
    String message,
    T data
) {
    public static <T> ApiResponse<T> success(T data) {
        return new ApiResponse<>(LocalDateTime.now(), 200, "Success", data);
    }

    public static <T> ApiResponse<T> error(int status, String message) {
        return new ApiResponse<>(LocalDateTime.now(), status, message, null);
    }
}
```

#### Step 3: Create Custom Processor

```java
public class CustomResponseBodyProcessor extends RequestResponseBodyMethodProcessor {

    public CustomResponseBodyProcessor(ObjectMapper objectMapper) {
        super(objectMapper);
    }

    @Override
    public boolean supports(MethodParameter returnType, ResponseBody annotation) {
        Method method = returnType.getMethod();
        if (method == null) return false;
        boolean hasMyResponseBody = method.isAnnotationPresent(MyResponseBody.class);
        boolean hasResponseBody = annotation != null;
        return hasMyResponseBody || hasResponseBody;
    }

    @Override
    public Object beforeBodyWrite(Object body, MethodParameter returnType, MediaType contentType,
                                   Class<? extends MappingJackson2HttpMessageConverter> converterType,
                                   ServerHttpRequest request, ServerHttpResponse response) {
        Method method = returnType.getMethod();
        if (method != null && method.isAnnotationPresent(MyResponseBody.class)) {
            if (body instanceof ApiResponse) return body;
            return ApiResponse.success(body);
        }
        return body;
    }
}
```

#### Step 4: Register the Processor

```java
@Configuration
public class WebConfig implements WebMvcRegistrations {
    @Override
    public RequestResponseBodyMethodProcessor getRequestResponseBodyMethodProcessor(ObjectMapper objectMapper) {
        return new CustomResponseBodyProcessor(objectMapper);
    }
}
```

#### Step 5: Use in Controller

```java
@MyResponseBody
@GetMapping("/profile")
public Map<String, String> profile() {
    return Map.of("name", "John", "email", "john@example.com");
}
```

### Response Example

```json
{
  "timestamp": "2026-04-08T12:00:00",
  "status": 200,
  "message": "Success",
  "data": { "name": "John", "email": "john@example.com" }
}
```

## 3. @ResponseStatus vs ResponseEntity

| Annotation                       | Use Case                                                  |
| -------------------------------- | --------------------------------------------------------- |
| `@ResponseStatus(HttpStatus.OK)` | Simple, static status codes with no dynamic response body |
| `ResponseEntity<T>`              | Dynamic status codes, headers, and response body          |
| `@MyResponseBody` (custom)       | When you need wrapped/transformed responses               |

## 4. Common Mistakes

1. **Using `@ResponseBody` with `@RestController`** - Redundant, `@RestController` already includes it
2. **Returning raw strings without wrapper** - Inconsistent API responses
3. **Not handling null bodies** - Causes serialization errors
4. **Forgetting `@Valid`** - Validation annotations are ignored without it on `@RequestBody`
