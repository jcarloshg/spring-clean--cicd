# Hibernate Implementation

## Overview

This document describes the Hibernate/JPA implementation in the `auth` package, following best practices for enterprise-grade persistence.

## Entity Design (`User.java`)

### Best Practices Applied

1. **`@DynamicUpdate`**: Generates UPDATE statements with only modified columns, optimizing performance for wide tables.

2. **`@NaturalId`**: Marks `email` as a natural business key. Hibernate optimizes lookups and caches these in a dedicated L2 cache region.

3. **UUID-based `equals()/hashCode()`**: Uses a UUID generated at instantiation instead of the database-generated `@Id` (which is null before persist). This ensures correct behavior with `Set` collections.

4. **Index on `email`**: Uses `@Index` annotation for the email column.

```java
@Entity
@Table(name = "users", indexes = {
    @Index(name = "idx_users_email", columnList = "email")
})
@DynamicUpdate
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @NaturalId
    @Column(name = "email", length = 255, nullable = false, unique = true)
    private String email;

    @Transient
    private final UUID uuid;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        User user = (User) o;
        return uuid.equals(user.uuid);
    }

    @Override
    public int hashCode() {
        return uuid.hashCode();
    }
}
```

## Repository Design (`UserRepository.java`)

- Extends `JpaRepository` for standard CRUD operations
- Uses `@Repository` annotation for exception translation (Spring converts JPA exceptions to Spring's exception hierarchy)
- Custom query methods using JPQL for type safety

```java
@Repository
public interface UserRepository extends JpaRepository<User, Long> {

    Optional<User> findByEmail(String email);

    boolean existsByEmail(String email);

    @Query("SELECT u FROM User u WHERE u.id = :id")
    Optional<User> findUserById(@Param("id") Long id);
}
```

## Transaction Management (`AuthServiceImpl.java`)

### Best Practices Applied

1. **Read-only transactions**: Uses `@Transactional(readOnly = true)` for query operations, allowing JPA/Hibernate optimizations.

2. **Write transactions**: Uses `@Transactional` for mutation operations.

3. **Short transactions**: Database operations are kept short; no I/O or heavy computation inside transactional boundaries.

```java
@Service
public class AuthServiceImpl implements AuthService {

    private final UserRepository userRepository;

    public AuthServiceImpl(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    @Override
    @Transactional(readOnly = true)
    public UserDto getById(Long id) {
        User user = userRepository.findUserById(id)
                .orElseThrow(() -> new UserNotFoundException(id));
        return toDto(user);
    }

    @Override
    @Transactional
    public UserDto signup(SignupRequest request) {
        if (userRepository.existsByEmail(request.email())) {
            throw new DuplicateEmailException(request.email());
        }
        User user = new User(request.name(), request.email(), request.pass());
        return toDto(userRepository.save(user));
    }
}
```

## DTO Projections

### Best Practice Applied

**DTO Projections for Read Operations**: Instead of returning managed entities from the service layer, we convert to DTOs. This:

- Bypasses the Hibernate Persistence Context (first-level cache)
- Reduces memory consumption
- Prevents accidental dirty writes
- Decouples API contract from persistence model

```java
public record UserDto(
    Long id,
    String name,
    String email,
    LocalDateTime createdAt
) {}

private UserDto toDto(User user) {
    return new UserDto(
        user.getId(),
        user.getName(),
        user.getEmail(),
        user.getCreatedAt()
    );
}
```

## Exception Handling

### Custom Exceptions

```java
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(Long id) {
        super("User not found with id: " + id);
    }
}

public class DuplicateEmailException extends RuntimeException {
    public DuplicateEmailException(String email) {
        super("User already exists with email: " + email);
    }
}
```

### Global Exception Handler

Handles both custom exceptions and JPA exceptions with appropriate HTTP status codes:

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<ApiResponse<Void>> handleUserNotFoundException(UserNotFoundException ex) {
        return ResponseEntity.status(HttpStatus.NOT_FOUND)
                .body(ApiResponse.error(HttpStatus.NOT_FOUND.value(), ex.getMessage()));
    }

    @ExceptionHandler(DuplicateEmailException.class)
    public ResponseEntity<ApiResponse<Void>> handleDuplicateEmailException(DuplicateEmailException ex) {
        return ResponseEntity.status(HttpStatus.CONFLICT)
                .body(ApiResponse.error(HttpStatus.CONFLICT.value(), ex.getMessage()));
    }

    @ExceptionHandler(DataIntegrityViolationException.class)
    public ResponseEntity<ApiResponse<Void>> handleDataIntegrityViolationException(DataIntegrityViolationException ex) {
        return ResponseEntity.status(HttpStatus.CONFLICT)
                .body(ApiResponse.error(HttpStatus.CONFLICT.value(), ex.getMostSpecificCause().getMessage()));
    }
}
```

## Key Hibernate Properties

Add to `application.properties`:

```properties
# Enable SQL logging for debugging
spring.jpa.show-sql=true

# Format SQL output
spring.jpa.properties.hibernate.format_sql=true

# Enable batch operations
spring.jpa.properties.hibernate.jdbc.batch_size=20
spring.jpa.properties.hibernate.order_inserts=true
spring.jpa.properties.hibernate.order_updates=true

# Enable dynamic update
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
```

## Summary of Best Practices

| Practice               | Implementation                                              |
| ---------------------- | ----------------------------------------------------------- |
| **Session Management** | Spring Data JPA handles session lifecycle via `@Repository` |
| **Lazy Loading**       | Default for `@OneToMany`; use `JOIN FETCH` when needed      |
| **N+1 Prevention**     | DTO projections bypass persistence context                  |
| **Transaction Scope**  | Short transactions, readOnly for queries                    |
| **DTO Projections**    | Convert entities to DTOs for read operations                |
| **Exception Handling** | Custom exceptions with `@RestControllerAdvice`              |
| **Equals/HashCode**    | UUID-based, not database ID                                 |
| **NaturalId**          | `@NaturalId` on email field                                 |
| **DynamicUpdate**      | Only modified columns in UPDATE                             |
| **Indexing**           | `@Index` annotation on frequently queried columns           |
